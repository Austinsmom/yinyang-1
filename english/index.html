YinYang

the client-side template framework written in CoffeeScript

  • Fork me on GitHub
  • Download 0.2 version

Truly for designers or HTML coders.
No more odd tags on preview.

No need for programing. HTML is directly generated by HTML files.

YinYang is a client side template and framework. It collates documents and templates as well as data retrieved by Ajax etc, and renders it as a single page via JavaScript. It doesn't need to be in a server side environment. The designer can focus on working on HTML without worrying about programming.

No more mockups needed. You can design on your template.

Do you think that creating mock-ups and creating templates are two separate things? It's true that in many template engines, the template isn't in HTML.
When you display the template in web browser strange symbols are displayed, or it only displays one when you need to see several side by side to get a sense of the image.
With YinYang, the template itself in perfect HTML, so you can modify it with a visual editor (Dreamweaver etc).

No database needed. Just place your docs on your folder.

Searching for files on a server is one of the things you can't do with Javascript. YinYang provides a SQL-like query language that is specialized for HTML file searches.

For example, if you need the three newest pages, just set it for
SELECT * FROM ./ LIMIT3.
*A server side PHP script environment is required to use the hSQL (html Search Query Language).
