YinYang

the client-side template framework written in CoffeeScript

  • Fork me on GitHub
  • Download 0.2 version

Truly for designers or HTML coders.
No more odd tags on preview.

No need for programing. HTML is directly generated by HTML files.

YinYang is a template framework that helps your client-side coding.
It renders a page by merging documents, templates, and data in background.
And it can run on any server-side environment. So, front-end engineers and designers are able to focus on coding HTML without worrying about how back-end systems runs.

No more mockups needed. You can design on your template.

Have you ever thought why you need to make mockups and templates separately? It's true that a template functions in a similar way to a html in most of template engines. However, rendering a template in web browser fails with showing not-familiar symbols with designers and HTML coders. So you might think if there were a way to render a template itself.
And here it is. With YinYang, the template functions not only a template in the engine, but also a html in any editors. So for example, you can directly render the template with a visual editor (DreamWeaver etc).

No database needed. Just place your docs on your folder.

Searching files in server is one of things that you can't do without Javascript. But YinYang provides you with a SQL-like language for searching such files.
To take an example, if you need the three newest pages, just type the following command.
SELECT * FROM ./ LIMIT 3
* You need to have server-side scripting environment (PHP) to use hSQL (html Search Query Language).
